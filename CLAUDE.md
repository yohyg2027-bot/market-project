# 마켓만들기 프로젝트

## 이 폴더는 무엇인가요?
개발 공부용 실습 폴더입니다.
현재 포함된 프로젝트:
- `가계부.html` — 가계부 웹앱 (이미 배포됨)
- `murder-help-market/` — 머더헬프마켓 (Next.js 중고거래 마켓, 개발 중)

---

## 기술 스택 (사용하는 도구들)

| 도구 | 쉬운 설명 |
|---|---|
| **Next.js** | 웹사이트의 뼈대. 페이지 이동, 로딩 속도, 서버 통신을 담당합니다 |
| **Supabase** | 구글 스프레드시트처럼 데이터를 저장하는 클라우드 데이터베이스 + 로그인 기능 포함 |
| **Vercel** | 만든 웹사이트를 인터넷에 올려주는 배포 서비스. GitHub에 올리면 자동으로 반영됩니다 |
| **TypeScript** | JavaScript에 오타 방지 기능을 추가한 언어 |
| **Tailwind CSS** | 미리 만들어진 디자인 도구 모음 |

## Claude가 지켜야 할 규칙

### 소통 방식
- **개발 용어는 항상 쉽게 풀어서 설명한다**
  - ❌ "RLS 정책을 적용합니다" → ✅ "다른 사람이 내 데이터를 못 보도록 보안 규칙을 설정합니다"
  - ❌ "미들웨어로 라우트를 보호합니다" → ✅ "로그인하지 않은 사람은 특정 페이지에 못 들어오게 막습니다"
  - ❌ "마이그레이션을 실행합니다" → ✅ "데이터베이스에 새 표(테이블)를 만듭니다"
- **작업 중에 지금 뭘 하고 있는지 자주 말해준다** ("지금 로그인 페이지를 만들고 있어요", "데이터베이스 연결 중입니다" 등)
- 완성 후에는 결과물이 어떻게 동작하는지 한 줄 요약

### 작업 방식
- **사용자가 지시한 것만 한다** — 요청하지 않은 기능을 미리 추가하지 않는다
- **단계적으로 진행한다** — 한 번에 하나씩, 확인받으며 진행
- **할 수 있는 건 최대한 직접 한다** — Claude가 할 수 있는 것은 묻지 않고 바로 처리
- **사람이 꼭 해야 하는 것만 안내한다** — 예: Vercel 배포, 이메일 인증 확인, 비밀키 발급 등

### 개발 완료 후
- **항상 미리보기를 열어서 확인할 수 있게 한다** (preview_start → preview_screenshot)
- 화면 캡처를 보여주며 "이렇게 만들어졌어요" 확인

## 사람이 직접 해야 하는 것들 (Claude가 대신 못 하는 것)

1. **Vercel 배포** — vercel.com에서 GitHub 연동 후 직접 버튼 클릭
2. **이메일 인증** — 회원가입 후 받은 이메일에서 인증 링크 클릭
3. **Supabase 비밀 키(service_role key)** — 대시보드에서 직접 복사
4. **도메인 구매/연결** — Vercel 설정에서 직접 진행
5. **소셜 로그인 앱 등록** — 구글/카카오 개발자 콘솔에서 직접 등록

## Supabase 프로젝트 정보

- **프로젝트 ID:** `pzikudakfsnahsyvtmbb`
- **지역:** 일본 도쿄 (ap-northeast-1) — 한국에서 빠르게 접속됩니다
- **현재 있는 테이블:**

  - `profiles` — 머더헬프마켓 사용자 정보 (닉네임, 이메일 등)
  - `products` — 게시글 (판매글 / 수요글 / 커뮤니티)
  - `comments` — 댓글 (비밀댓글 포함)
  - `likes` — 게시글 좋아요
  - `chats` — 1:1 채팅방
  - `messages` — 채팅 메시지 (실시간)
